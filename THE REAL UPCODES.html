Z80 instruction set

This is an overview of the Z80 instruction set. Undocumented instructions are included (note that Sean Young wrote an extensive document about them), and the R800 instructions MULUB and MULUW as well as R800 timing information.

Important note: the timings in this list are excluding the so-called M1 waits specific to the MSX. What happens is that the MSX engine adds one additional wait state to every M1 cycle (the first or first two cycles of an instruction). This is an important trait to know about if you want to do speed calculations, for example. The M1 waits are inserted like this:

  • Every instruction has 1 additional M1 wait state
  • Instructions starting with a CB, DD, ED, FD, DDCB or DDFD opcode have two additional M1 wait states

Examples:

  • XOR A has opcode AF and is documented as taking 4 T-states. On MSX, it takes 5 (one more).
  • BIT 0,A has opcode CB 47 and is documented as taking 12 T-states. On MSX, it takes 14 (two more).
  • SRL (IX+0) has opcode DD CB 00 3E and is documented as taking 23 T-states. On MSX, it takes 25 (two more).

For the R800, it is an even more complex story (Warning: preliminary info! To be frank, this is just a guess/indication. I need to verify!):

  • In internal RAM, 0 wait states are inserted, and an additional wait state for every memory access where the MSB changes (far).
  • In external RAM, 2 wait states are inserted.
  • In ROM, 3 wait states are inserted.
  • For every far memory fetch, additional wait states are inserted depending on the memory type.
